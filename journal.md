# 초기 기획 단계라 앱 개발 과정에서 많은 것이 변경되었음


# 1. 문제 정의: 왜 이 앱을 만드는가? (구체적으로)
급식에서 맛있는 음식에 가중치를 붙여서 정렬하는 앱
급식 정보를 **나이스 대국민서비스: 급식 api(무료)** 로 불러온다.
불러온 정보를 **구글 제미나이 api(무료)** 로 가공한다.


# 2. 사용자 시나리오: 누가, 언제 사용하는가?
급식 정보를 깔끔하게 보고 싶은 학생들


# 3. 필요한 위젯: 어떤 버튼과 입력창이 필요한가?
날짜 선택 UI, 불러오기 버튼


# 4. 화면 스케치 (텍스트): [상단: 제목] -> [중단: 점수 슬라이더] -> [하단: 결과 메시지] 식의 흐름도 작성
<맨위> 날짜 선택 UI (또는 날짜 입력하는 입력창)
<중간> 불러오기 버튼
<하단> 정보가 정제된 급식 데이터 불러오기
